FA2M — E-commerce (Django API + React PWA + Capacitor Android)
================================================================

Prérequis : Python 3.11+, Node 18+, (optionnel) PostgreSQL, Android Studio pour l’APK.

BACKEND (Django)
----------------
  cd backend
  python -m venv .venv
  .venv\Scripts\activate          (Windows)
  pip install -r requirements.txt
  copy .env.example .env          (ajuster SECRET_KEY, CORS, email…)
  python manage.py migrate
  python manage.py createsuperuser
  python manage.py runserver

API : http://127.0.0.1:8000/api/  — Admin Django : http://127.0.0.1:8000/admin/

Base de données : SQLite par défaut. Pour PostgreSQL, définir POSTGRES_* dans .env (voir .env.example).

FRONTEND (React + Vite + MUI + PWA)
------------------------------------
  cd frontend
  npm install
  npm run dev

Interface : http://127.0.0.1:5173 — le proxy Vite envoie /api et /media vers le backend local.

Production / APK : définir VITE_API_URL (URL absolue de l’API, ex. https://votredomaine.com/api) dans
frontend/.env puis build :

  npm run build
  npx cap add android          (une première fois)
  npx cap sync
  npx cap open android

Ouvrir le projet Android dans Android Studio, générer un APK ou App Bundle (signed).

NOTES
-----
- Paiements : livraison (cod) et retrait (pickup) actifs ; Stripe / PayPal / Mobile Money prévus dans le modèle (réponse 501 si sélectionnés sans implémentation).
- Notifications push : FCM — brancher firebase-admin côté Django et enregistrer les tokens (POST /api/auth/fcm-token/).
- Ancienne vitrine statique : index.html, styles.css, script.js à la racine du dépôt (inchangée).
